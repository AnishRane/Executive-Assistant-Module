What each capture feeds

1 Edit brief Diff between agent draft and David’s edit. Feeds david.voice. “Removed two superlatives, shortened the second sentence by 40%” is a stronger tone signal than any rating could be.
2 Edit attendee bio Manual correction to what the agent inferred from a signature. Feeds people.<email> directly. The agent stops re-inferring on later meetings.
3 Pin another thread David pins a chain the agent missed. Logs an email_anchors row + a pinned_thread feedback signal. Trains the agent on what counts as “in scope” for similar future meetings.
4 Dismiss open item The agent guessed wrong about an obligation. Sets action_items.status = ignored and writes a dismissed_action signal. Over time, this calibrates how cautious the action-item extractor is.
5 “Less of this” on a delta David doesn’t care to be told about meeting time shifts under 30 minutes (or about Acme specifically, depending on what the agent infers). Writes a marked_unhelpful signal. Eventually feeds the elevation policy.
6 “Take this · ask <X> to swap” A conflict resolution. Records a row in a new executive_assistant__conflicts table marking which item David chose and which one is being deprioritised, plus a conflict_resolved feedback signal carrying the choice. Over time, this feeds a memory like “board‑prep blocks are dispensable when client calls collide.”
